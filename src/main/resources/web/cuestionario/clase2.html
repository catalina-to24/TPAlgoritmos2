Complejidad algorítmica

Cómo medir qué tan eficiente es un algoritmo sin ejecutarlo: conteo de instrucciones, notación Big O y tasas de crecimiento. Este tema pesa 30% de la nota del TPO.

T(n) Big O Asintótica Demostración formal

1 ¿Qué es un buen algoritmo?

Hay criterios subjetivos (claridad, simplicidad, adecuación a los datos) y uno más objetivo: la eficiencia en tiempo de ejecución. No sirve medir en segundos porque dependen del hardware; en vez de eso contamos instrucciones en función del tamaño de la entrada n.

2 Tiempo de ejecución T(n)

T(n) es el número de instrucciones ejecutadas en un "computador idealizado" para una entrada de tamaño n. Conteo de ejemplo:

boolean todosPares(int[] arr) {
    boolean rta = true;                            // 1
    for (int i = 1; i < arr.length; i++) {         // 1 + 2(n+1) + n = 3 + 3n
        rta = rta && (arr[i] % 2) == 0;           // 5n
    }
    return rta;                                    // 1
}

f(n) = 1 + 3 + 3n + 5n + 1 = 8n + 5

El resultado exacto depende de cómo contás (cada operación cuesta 1, cada comparación cuesta 1, etc.). Lo importante es el grado del polinomio, no la constante.

3 Complejidad asintótica

Interesa cómo crece el costo cuando n tiende a infinito: se descartan constantes multiplicativas y términos de menor orden. Es una cota (en general superior), no un tiempo exacto.

¿Por qué asintótica? Para n chico cualquier algoritmo parece rápido. La diferencia entre O(n) y O(n²) se nota con entradas grandes, que son las que realmente estresan el sistema.

4 Notación O (Big O)

Se dice que f(n) ∈ O(g(n)) si existen constantes c > 0 y n₀ ≥ 0 tales que:

f(n) ≤ c · g(n) para todo n ≥ n₀

Leído: "el costo de f(n) nunca supera a un múltiplo de g(n) para n suficientemente grande".

Demostración formal que f(n) = 8n + 5 es O(n)

  1. Elegir g(n): queremos probar que f(n) ∈ O(n), así que g(n) = n.
  2. Planteo: buscamos c y n₀ tales que 8n + 5 ≤ c·n.
  3. Dividimos por n (con n > 0): 8 + 5/n ≤ c.
  4. Elegimos c = 9. Entonces 8 + 5/n ≤ 9 ⇒ 5/n ≤ 1 ⇒ n ≥ 5.
  5. Tomamos n₀ = 5. Para todo n ≥ 5 se cumple. QED.

Conclusión: f(n) = 8n + 5 es O(n) con c = 9 y n₀ = 5.

6 Ejercicio típico: matriz cuadrada

Calcular el promedio de una matriz n × n:

int[][] mat = ...;
int suma = 0;                                              // 1
for (int i = 0; i < mat.length; i++) {                     // 3 + 3n
    for (int j = 0; j < mat[0].length; j++) {              // 3 + 3n
        suma += mat[i][j];                                 // 2n
    }
}
double promedio = (double) suma / (mat.length * mat[0].length);  // 2
System.out.println("Promedio " + promedio);                // 2

f(n) = 6 + 6n + 5n²

Demostración que pertenece a O(n²): buscamos c tal que 6 + 6n + 5n² ≤ c·n². Dividiendo por n²: 6/n² + 6/n + 5 ≤ c. Para n = 1: 6 + 6 + 5 = 17 ≤ c. Tomando c = 17 y n₀ = 1 queda probado.

7 Tips rápidos para el parcial

  1. Un for que recorre toda la colección es O(n).
  2. Dos for anidados sobre la misma colección son O(n²).
  3. Operaciones que reducen el problema a la mitad cada vez son O(log n).
  4. La constante no importa: O(5n) se escribe O(n).
  5. El término de menor orden no importa: O(n² + n) se escribe O(n²).
  6. Big O es cota superior: está pensada para acotar el peor caso.
