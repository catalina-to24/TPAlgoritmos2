TDAs: Par Ordenado, Tupla y Conjunto

Primera clase fuerte del primer parcial: qué es un Tipo de Dato Abstracto, cómo se especifica, qué es un invariante de representación y las tres primeras implementaciones en Java con genéricos.

TDA Invariante Genéricos <T> POO

1 ¿Qué es un TDA?

Un Tipo de Dato Abstracto (TDA, en inglés ADT) define una estructura en términos de su comportamiento y operaciones, sin especificar la implementación interna. Dice qué hace, no cómo lo hace.

Ejemplos comunes:

  • Lista: agregar, eliminar, obtener(i) — se puede implementar con array, lista enlazada, doblemente enlazada, etc.
  • Cola: encolar, desencolar — array circular, lista enlazada, etc.
  • Pila: apilar, desapilar — array, lista enlazada, etc.
💡 Idea fundamental Encapsular los detalles y ofrecer una interfaz limpia. Así podés cambiar la implementación sin romper el código que usa el TDA.

2 Invariante de representación

Una condición que siempre debe cumplirse para que un objeto del TDA esté en estado válido. Se chequea antes y después de cada operación.

Ejemplos:

  • Lista enlazada: el último nodo apunta a null; cada nodo apunta al siguiente.
  • Cola: los elementos se agregan al final y se retiran del frente (FIFO).
  • Fecha: día, mes y año están en rangos válidos y son coherentes.
  • Número complejo: parte imaginaria es un real válido.

El invariante de representación es lo que garantiza la integridad de los datos. Violarlo rompe el TDA.

3 POO rápida para implementar TDAs

  • Clase: define el molde de los objetos. Se declara con class.
  • Objeto: instancia de una clase. Se crea con new.
  • Atributos (propiedades): almacenan el estado.
  • Métodos: definen el comportamiento.
  • Genéricos <T, U>: permiten que una clase funcione con cualquier tipo sin duplicar código.

4 TDA Par Ordenado

Representa dos elementos donde el orden importa: (a, b) ≠ (b, a) salvo que se defina explícitamente como simétrico. Formalmente (Kuratowski): (a, b) = { {a}, {a, b} }.

Operaciones

OperaciónEntrada → Salida
CrearPar(a, b)→ ParOrdenado
ObtenerPrimero(par)→ TipoDato (el a)
ObtenerSegundo(par)→ TipoDato (el b)
EsIgual((a,b), (c,d))→ boolean (true si a=c y b=d)

Invariantes de representación

  • El par siempre contiene dos elementos.
  • El par no debe ser nulo.
  • El orden es relevante.
  • El par es inmutable.

Implementación en Java con genéricos

public class ParOrdenado<T, U> {
    private final T primero;
    private final U segundo;

    public ParOrdenado(T primero, U segundo) {
        this.primero = primero;
        this.segundo = segundo;
    }

    public T obtenerPrimero() { return primero; }
    public U obtenerSegundo() { return segundo; }

    @Override
    public String toString() {
        return "(" + primero + ", " + segundo + ")";
    }
}

Aplicaciones

  • Coordenadas en un plano: (x, y).
  • Clave-valor en diccionarios.
  • Aristas de un grafo: (nodo1, nodo2).

5 TDA Tupla

Generalización del Par Ordenado: colección ordenada de n elementos, usualmente de distintos tipos. En el curso se implementan Tupla2, Tupla3, Tupla4.

Operaciones

  • ObtenerElemento(T, i) → valor en posición i.
  • Tamaño(T) → n
  • Concatenar(T1, T2) → nueva tupla
  • EsIgual(T1, T2) → boolean
  • Desempaquetar(T) → (x1, x2, ..., xn)

Invariantes

  1. Cantidad fija de elementos. No se puede agregar ni quitar.
  2. Orden preservado.
  3. Inmutable: toda "modificación" devuelve una tupla nueva.
public final class Tupla3<T, U, V> {
    private final T primero;
    private final U segundo;
    private final V tercero;

    public Tupla3(T p, U s, V t) { primero=p; segundo=s; tercero=t; }
    public T getPrimero() { return primero; }
    public U getSegundo() { return segundo; }
    public V getTercero() { return tercero; }
}

6 TDA Conjunto

Colección sin orden específico y sin elementos duplicados. Solo interesa la pertenencia.

Notación: C = { e₁, e₂, ..., eₙ }, todos del mismo dominio.

Operaciones

OperaciónQué hace
crear()Crea un conjunto vacío.
agregar(e)Inserta e si no está presente.
eliminar(e)Elimina e si está presente.
pertenece(e)→ true si e ∈ C.
esVacio()→ true si no hay elementos.
cardinalidad()→ cantidad de elementos.
union(C₂)→ C ∪ C₂.
interseccion(C₂)→ C ∩ C₂.
diferencia(C₁, C₂)→ C₁ \ C₂.
esSubconjunto(C₁, C₂)→ true si C₁ ⊆ C₂.

Invariantes

  1. No hay duplicados: ∀ eᵢ, eⱼ ∈ C, si i ≠ j entonces eᵢ ≠ eⱼ.
  2. El conjunto vacío es un caso válido: C = ∅.
  3. Los elementos pertenecen al dominio definido.

Implementaciones típicas

  • Estática con array de tamaño fijo.
  • Dinámica con lista enlazada.
  • En Java estándar: HashSet<E> (hash), TreeSet<E> (árbol balanceado, ordenado).
⚠ Complejidad de agregar En una implementación naive con lista enlazada, agregar(e) es O(n) porque hay que recorrer toda la lista para verificar que no esté. Si insertás al principio sin chequear duplicados es O(1), pero rompés el invariante.

7 Puntos clave para el examen

  1. Saber definir qué es un TDA en una frase: "estructura especificada por comportamiento, no implementación".
  2. Poder enumerar los invariantes de cada TDA visto: Par Ordenado, Tupla, Conjunto.
  3. Diferencia Par Ordenado vs Tupla: el primero siempre tiene 2 elementos; la tupla tiene n.
  4. Diferencia Conjunto vs Lista: el conjunto no mantiene orden y no admite duplicados.
  5. Genéricos en Java: class ParOrdenado<T, U> para trabajar con cualquier tipo sin duplicar código.
  6. Inmutabilidad: campos final + constructor + getters, sin setters.
📂 En el TPO: Paquete<T> es un ejemplo de clase genérica e inmutable (todos los campos final, igualdad por ID). Ruta es un Par Ordenado matizado (con distancia) implementado como record.
