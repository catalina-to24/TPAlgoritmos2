TADs específicos del dominio, archivos y JSON

Última clase antes del parcial: cómo modelar entidades de un problema concreto como TDAs (Cliente, Producto, etc.) y cómo persistir/leer datos usando archivos y formato JSON con Gson.

TAD del dominio Archivos JSON Gson

1 TADs específicos del dominio

A diferencia de los TDAs generales (lista, pila, cola, diccionario), los TDAs del dominio modelan entidades concretas de un problema específico: cliente, producto, paquete, depósito, empleado, etc.

Estructura común

  1. Datos: atributos esenciales de la entidad.
  2. Operaciones: acciones que tienen sentido sobre la entidad.
  3. Invariante de representación: restricciones para que el estado sea válido.

2 Ejemplo 1: TAD Cliente

ParteContenido
Datos Nombre, número de identificación (DNI/ID), historial de compras o reservas.
Operaciones Crear un cliente con nombre e ID; agregar una compra; consultar historial.
Invariante El número de identificación no debe ser nulo.

3 Ejemplo 2: TAD Producto

ParteContenido
Datos Código, nombre, precio, stock disponible.
Operaciones Registrar nuevo producto; modificar stock; calcular precio con impuestos.
Invariante El precio es positivo. El stock no puede ser negativo.

4 Ejercicio típico: TecnoShop

Sistema que registra clientes y su historial de compras. Objetivo: identificar los TDAs necesarios.

  • Cliente — ID único, nombre, historial de compras.
  • Compra — producto adquirido, fecha.
  • Un cliente tiene una lista de compras.
  • El sistema usa un diccionario (clave = ID del cliente, valor = Cliente) para ubicarlos rápido.
💡 Patrón a reconocer Los TDAs del dominio usan los TDAs generales como piezas de lego: un Cliente contiene una Lista de Compra. Saber combinar TDAs es la esencia del parcial.

5 Manejo de archivos en Java

Lectura línea por línea

try {
    FileReader reader = new FileReader("texto.txt");
    BufferedReader bufferedReader = new BufferedReader(reader);
    String line;
    while ((line = bufferedReader.readLine()) != null) {
        System.out.println(line);
    }
    reader.close();
} catch (IOException e) {
    e.printStackTrace();
}

Escritura (append)

try {
    FileWriter fileWriter = new FileWriter("texto.txt", true); // true = append
    fileWriter.write("\n mas texto");
    fileWriter.close();
} catch (Exception e) {
    e.printStackTrace();
}
⚠ Siempre en try/catch Las operaciones de archivo pueden lanzar IOException. Si no las envolvés en try/catch el código no compila.

6 Formato JSON

JSON (JavaScript Object Notation) es un formato ligero de intercambio de datos. Es el estándar de facto en la web.

  • Objeto: pares clave: valor entre llaves {}.
  • Array: lista de valores entre corchetes [].
  • Valores válidos: string, número, boolean, null, objeto, array.
  • Claves siempre son strings.
{
  "nombre": "Juan Pérez",
  "edad": 30,
  "ciudad": "Ejemploville",
  "casado": false,
  "intereses": ["programación", "viajes", "lectura"]
}

7 Librería Gson

Gson (Google) convierte objetos Java ↔ JSON con dos llamadas. Se descarga como gson-2.8.2.jar y se agrega a libs/.

De JSON a objeto

Gson gson = new Gson();
try {
    FileReader reader = new FileReader("datosjson.txt");
    Sistema sistema = gson.fromJson(reader, Sistema.class);
    System.out.println(sistema);
} catch (Exception e) {
    e.printStackTrace();
}

De objeto a JSON y escribirlo

Gson gson = new Gson();
try {
    Resultado resultado = new Resultado("Cantidad de ventas " + sistema.cantidadVentas());
    String json = gson.toJson(resultado);
    FileWriter fileWriter = new FileWriter("resjson.txt");
    fileWriter.write(json);
    fileWriter.close();
} catch (Exception e) {
    e.printStackTrace();
}
💡 Cómo funciona fromJson(reader, Clase.class) usa reflexión para instanciar la clase y asignar cada campo desde el JSON. Los nombres de campo tienen que coincidir (o anotarse con @SerializedName).

8 Puntos clave para el examen

  1. Un TAD del dominio tiene: datos, operaciones, invariantes. Saber enunciar los tres.
  2. Los TDAs del dominio combinan los generales: un Cliente puede tener una Lista de Compras.
  3. Lectura de archivo: FileReader + BufferedReader + readLine() en loop.
  4. Escritura de archivo: FileWriter con true para modo append.
  5. JSON: claves strings entre comillas, objetos con {}, arrays con [].
  6. Gson: fromJson(reader, Clase.class) / toJson(objeto).
  7. Todas las operaciones de IO van dentro de try/catch.

El cargador (InventarioLoader.java) y el parser casero (SimpleJsonParser.java) hacen lo que haría Gson, pero implementados a mano para cumplir el requisito de "sin dependencias externas". Si tu equipo usó Gson, el flujo es idéntico.
