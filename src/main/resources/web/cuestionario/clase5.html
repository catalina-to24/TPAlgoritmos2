TAD Deque y TAD Diccionario

Dos TDAs más sofisticados: el Deque es una cola de doble terminación (puede actuar como pila o cola); el Diccionario asocia claves únicas con valores.

Deque Diccionario Clave-Valor Hash/Tree

1 TAD Deque (Double-Ended Queue)

Cola doblemente terminada: permite insertar y eliminar por ambos extremos. Puede usarse como pila (solo un extremo) o como cola (encolar atrás, desencolar adelante).

Operaciones

OperaciónQué haceComplejidad
agregarInicio(e)Inserta al frente.O(1)
agregarFinal(e)Inserta al final.O(1)
eliminarInicio()Remueve y devuelve el primero.O(1)
eliminarFinal()Remueve y devuelve el último.O(1)
verInicio()Primer elemento sin eliminar.O(1)
verFinal()Último sin eliminar.O(1)
estaVacio()→ boolean.O(1)
tamaño()Cantidad de elementos.O(1)

Invariantes

  1. El orden de los elementos se mantiene según cómo se insertaron y retiraron.
  2. Las operaciones en ambos extremos son O(1) en implementaciones óptimas (lista doblemente enlazada o array circular).
  3. Dependiendo de la implementación, puede prohibir null.

Implementaciones en Java

  • LinkedList<E> — lista doblemente enlazada, implementa Deque<E>.
  • ArrayDeque<E> — recomendada. Array redimensionable, no permite null, más rápida que LinkedList.
💡 Usos típicos Historial de navegador (atrás/adelante), BFS que requiere insertar/sacar en ambos extremos, colas con prioridades binarias (alta al frente, baja al final).

Ejercicio típico: prioridad simple con deque

Tareas urgentes van al inicio (agregarInicio) y se atienden primero; tareas normales van al final (agregarFinal). Se procesa siempre con eliminarInicio().

2 TAD Diccionario (Mapa / Tabla de asociación)

Estructura que asocia claves con valores. Permite buscar un valor a partir de su clave en tiempo eficiente. También se llama Mapa o Tabla de asociación.

Notación formal: D = { (k₁, v₁), (k₂, v₂), ..., (kₙ, vₙ) }, con claves únicas.

Operaciones

OperaciónQué hace
insertar(k, v)Agrega el par (k, v). Si la clave ya existe, actualiza el valor.
obtener(k)Devuelve el valor asociado a k, o null / error si no existe.
eliminar(k)Quita la clave y su valor.
contiene(k)→ true si la clave está presente.
size()Total de pares.
claves()Conjunto con todas las claves.
valores()Lista con todos los valores.

Invariantes de representación

  1. Las claves son únicas: no puede haber dos pares con la misma clave.
  2. Cada clave tiene exactamente un valor asociado.
  3. Los valores pueden repetirse; las claves no.
  4. Puede estar vacío, pero generalmente no admite clave null.

Implementaciones en Java

ClaseEstructura internaOrdenComplejidad típica
HashMap<K,V>Tabla hashNo ordenadoO(1) promedio
TreeMap<K,V>Árbol balanceado (red-black)Por clave ascendenteO(log n)
LinkedHashMap<K,V>Hash + lista doblePor orden de inserciónO(1) promedio
⚠ Cuándo elegir cuál HashMap si necesitás velocidad y no te importa el orden.
TreeMap si necesitás recorrer las claves ordenadas.
LinkedHashMap si querés preservar el orden de inserción (útil para caches LRU).

Ejercicio típico: gestión de estudiantes

  • Clave: legajo (entero).
  • Valor: Tupla con nombre, apellido y lista de calificaciones.
  • Operaciones: agregar estudiante, agregar nota, promedio por estudiante, listar todos.

3 Comparación Deque vs Pila vs Cola

PilaColaDeque
AccesoSolo cimaSolo frente/finalAmbos extremos
Insertar al inicio——✓ O(1)
Insertar al final✓ (cima)✓ (encolar)✓ O(1)
Eliminar del inicio—✓ (desencolar)✓ O(1)
Eliminar del final✓ (desapilar)—✓ O(1)

El Deque es la generalización de Pila y Cola: con sólo dos de sus cuatro operaciones podés simular cualquiera.

4 Puntos clave para el examen

  1. Un Deque permite operar en ambos extremos. Todas las operaciones son O(1) en buena implementación.
  2. Un Diccionario mapea claves únicas a valores.
  3. Las claves no se repiten; los valores sí pueden.
  4. En Java: HashMap es O(1) promedio pero no ordenado; TreeMap es O(log n) pero ordenado por clave.
  5. Un HashSet es básicamente un HashMap sin valores (solo claves).
  6. Los operadores del Deque son: agregarInicio, agregarFinal, eliminarInicio, eliminarFinal, verInicio, verFinal.
📂 En el TPO: Camion.java usa ArrayDeque internamente porque la JDK recomienda esa clase para pilas. El TPO no usa diccionario explícitamente, pero los loaders (SimpleJsonParser) devuelven LinkedHashMap para preservar el orden de inserción del JSON.
